# DeckLab 2.0 上位机代码说明文档

## 1. 项目概述

DeckLab 2.0 是一个基于 Qt Widgets 的六自由度平台上位机控制程序，主要用于同时管理 Stewart 六自由度运动平台、本地扩展伺服轴以及 BRT38 SSI 拉线位移传感器数据。

程序采用双通信链路设计：

- UDP 链路：用于与国辰正域 Stewart 六自由度平台控制器通信，负责平台上电、下电、故障复位、运动模式设置、目标位姿发送以及实时状态接收。
- Modbus TCP 链路：用于与汇川 AC702/PLC 通信，负责本地扩展轴控制和 BRT38 SSI 拉线传感器数据读取。

主界面采用三栏式布局：

- 左侧：本地扩展轴控制面板 `MotorPanel`
- 中间：六自由度平台控制面板 `PlatformPanel`
- 右侧：拉线位移传感器显示面板 `SensorPanel`

## 4. 总体架构

系统按 UI 层、Worker 层和通信层组织。

```text
MainWindow
├── MotorPanel       ──signals──> PlcWorker ──> ModbusTcpClient ──> PLC
├── SensorPanel      <─signals── PlcWorker <── ModbusTcpClient <── PLC
└── PlatformPanel    ──signals──> PlatformWorker ──> UdpPlatformClient ──> Stewart 控制器
                    <─signals── PlatformWorker <── UdpPlatformClient <── Stewart 控制器
```

设计原则：

- UI 线程只负责显示和用户输入，不直接执行阻塞通信。
- Modbus TCP 和 UDP 通信分别运行在独立 worker 中。
- UI 与 worker 之间通过 Qt 信号槽连接，跨线程连接使用 `Qt::QueuedConnection`。
- 实时数据由 worker 读取或接收后推送到对应 UI 面板。

## 5. 主窗口模块

对应文件：

- `mainwindow.h`
- `mainwindow.cpp`

`MainWindow` 负责程序主框架和线程生命周期管理。

主要职责：

1. 创建主界面。
2. 创建顶部连接配置区。
3. 创建三栏式主工作区。
4. 管理 PLC worker 线程。
5. 管理平台 UDP worker 线程。
6. 连接 UI 信号和 worker 槽函数。
7. 在连接状态变化时同步刷新各面板状态。

### 5.1 顶部连接区

顶部连接区分为两段：

- PLC / Modbus TCP：IP、端口、从站号、连接按钮、连接状态。
- Stewart / UDP：IP、命令端口、连接按钮、连接状态。

默认参数：

| 链路 | 默认 IP | 默认端口 |
| --- | --- | --- |
| PLC Modbus TCP | `192.168.1.1` | `502` |
| Stewart UDP | `192.168.1.88` | `2000` |

### 5.2 三栏布局

主窗口使用 `QSplitter` 做水平三栏布局。

| 位置 | 面板 | 功能 |
| --- | --- | --- |
| 左侧 | `MotorPanel` | 卷筒电机和 RSD 丝杆控制 |
| 中间 | `PlatformPanel` | Stewart 六自由度平台控制 |
| 右侧 | `SensorPanel` | 6 路 BRT38 SSI 拉线传感器数据显示 |

## 6. PLC / Modbus TCP 通信模块

对应文件：

- `communication/ModbusTcpClient.h`
- `communication/ModbusTcpClient.cpp`
- `communication/PlcWorker.h`
- `communication/PlcWorker.cpp`
- `communication/RegisterMap.h`

### 6.1 ModbusTcpClient

`ModbusTcpClient` 封装底层 Modbus TCP 报文发送和接收，基于 `QTcpSocket` 实现。

主要接口：

| 接口 | 说明 |
| --- | --- |
| `connectToPlc()` | 连接 PLC |
| `disconnectFromPlc()` | 断开 PLC |
| `isConnected()` | 查询连接状态 |
| `readHoldingRegisters()` | 读取保持寄存器 |
| `writeSingleRegister()` | 写单个寄存器 |
| `writeMultipleRegisters()` | 写多个寄存器 |

### 6.2 PlcWorker

`PlcWorker` 运行在独立 `QThread` 中，是 PLC 通信和数据轮询的核心。

主要职责：

- 初始化 `ModbusTcpClient`。
- 20 ms 定时轮询 PLC，理论刷新率 50 Hz。
- 读取本地扩展轴状态。
- 读取 6 路拉线位移传感器数据。
- 接收 UI 发来的电机控制命令。
- 断线后自动重连，重连间隔从 500 ms 开始指数退避，上限 10 s。

### 6.3 PLC 轮询流程

```text
QTimer timeout
└── PlcWorker::pollCycle()
    ├── 检查 Modbus 连接状态
    ├── readAllMotors()
    └── readSensors()
```

如果连接断开：

1. 停止轮询定时器。
2. 发送 `connectionStateChanged(false)`。
3. 启动重连定时器。
4. 重连成功后清空本地轴控制字并恢复轮询。

## 7. 本地扩展轴控制模块

对应文件：

- `motor/MotorController.h`
- `motor/MotorController.cpp`
- `motor/MotorPanel.h`
- `motor/MotorPanel.cpp`

### 7.1 轴分配

程序中本地扩展轴为 PLC 的第 7、8 轴，对应 Qt 内部轴号 6、7。

| Qt 轴号 | PLC 轴 | 功能 |
| --- | --- | --- |
| 6 | Axis_7 | 卷筒电机 |
| 7 | Axis_8 | RSD 丝杆 |

### 7.2 MotorPanel

`MotorPanel` 是左侧电机控制 UI，包含两个轴的控制卡片。

每个轴显示：

- 当前状态
- 实际速度
- 实际位置
- 当前转矩
- 故障码

每个轴支持：

- 使能
- 去使能
- 故障复位
- 速度发送
- 位置定位

面板底部提供全局急停按钮，触发 `emergencyStopAllRequested()` 信号。

### 7.3 电机控制命令

UI 发出的控制信号最终进入 `PlcWorker`：

| UI 信号 | Worker 槽函数 | 说明 |
| --- | --- | --- |
| `enableAxisRequested` | `enableAxis` | 写控制字使能 |
| `disableAxisRequested` | `disableAxis` | 写控制字去使能 |
| `faultResetRequested` | `faultResetAxis` | 执行故障复位序列 |
| `setVelocityRequested` | `setVelocity` | 写目标速度并触发运行 |
| `setPositionRequested` | `setPosition` | 写目标位置和接近速度 |
| `emergencyStopAllRequested` | `emergencyStopAll` | 所有轴写停止控制字 |

### 7.4 数据结构 MotorData

`MotorData` 保存单轴的实时状态。

主要字段：

| 字段 | 说明 |
| --- | --- |
| `axisId` | Qt 内部轴号 |
| `statusWord` | PLC 读回的状态字 |
| `actualVel` | 实际速度，程序中按 `/10.0` 显示 rpm |
| `actualPos` | 实际位置，32 位 pulse |
| `torque` | 转矩，程序中按 `/10.0` 显示百分比 |
| `errorCode` | 驱动器故障码 |
| `enabled` | 是否已使能 |
| `faultActive` | 是否处于故障状态 |
| `stateName` | 状态文字 |

## 8. 六自由度平台 UDP 通信模块

对应文件：

- `platform/PlatformData.h`
- `platform/UdpPlatformClient.h`
- `platform/UdpPlatformClient.cpp`
- `platform/PlatformWorker.h`
- `platform/PlatformWorker.cpp`
- `platform/PlatformPanel.h`
- `platform/PlatformPanel.cpp`

### 8.1 PlatformData

`PlatformData.h` 定义平台相关数据结构。

| 类型 | 说明 |
| --- | --- |
| `PlatformConfig` | UDP 通信配置，包含 IP、命令端口、推送端口 |
| `PlatformPose` | 六自由度位姿，包含 X/Y/Z/Rx/Ry/Rz |
| `MotionMode` | 平台运动模式枚举 |
| `PlatformState` | 平台实时状态快照 |
| `FuncCode` | UDP 协议功能码定义 |

### 8.2 UDP 协议

命令帧：

| 项 | 内容 |
| --- | --- |
| 帧长 | 500 字节 |
| 帧头 | `0xAA55` |
| 请求类型 | `1001` 读，`2001` 写 |
| 数据区 | 功能码 + 保留字段 + double 数值 |
| 帧尾 | `0x66BB` |

推送帧：

| 项 | 内容 |
| --- | --- |
| 帧长 | 1578 字节 |
| 方向 | 平台控制器推送到上位机 |
| 推送端口 | 默认 `3001` |
| 内容 | 功能码索引 + double 数据区 |

### 8.3 关键功能码

| 功能码 | 含义 |
| --- | --- |
| `10000` | 平台上电/下电 |
| `10001` | 故障复位 |
| `10010` | 运动模式 |
| `10300` - `10305` | 目标位姿 X/Y/Z/Rx/Ry/Rz |
| `20000` | 设备状态字 |
| `21000` - `21005` | 当前位姿 X/Y/Z/Rx/Ry/Rz |
| `20200` - `20205` | 6 条支链实际长度 |
| `22100` - `22105` | 外部传感器数据 |

### 8.4 PlatformWorker

`PlatformWorker` 运行在独立 `QThread` 中，负责调用 `UdpPlatformClient`。

主要职责：

- 绑定 UDP socket。
- 接收平台推送状态。
- 将 UI 控制命令转换为 UDP 功能码写入。
- 通过看门狗判断平台是否在线。

看门狗逻辑：

- 定时周期：200 ms。
- 连续 5 次未收到推送帧，即约 1 s 未收到数据，则认为平台离线。
- 收到有效推送后重新标记在线。

### 8.5 PlatformPanel

`PlatformPanel` 是中间六自由度平台 UI。

主要显示：

- UDP 在线/离线状态
- 电源状态
- 当前运动模式
- 故障状态
- 当前位姿 X/Y/Z/Rx/Ry/Rz
- 支链长度 L1-L6

主要控制：

- 上电
- 下电
- 故障复位
- 设置运动模式
- 发送目标位姿
- 停止/急停

## 9. 拉线传感器模块

对应文件：

- `sensor/SensorData.h`
- `sensor/SensorPanel.h`
- `sensor/SensorPanel.cpp`
- `communication/PlcWorker.cpp`

### 9.1 SensorData

`SensorData.h` 定义了传感器数据结构。

```cpp
struct SensorChannel {
    QString name;
    QString unit;
    qint32 rawCounts;
    float displayValue;
    bool valid;
};

struct SensorData {
    static constexpr int COUNT = 6;
    std::array<SensorChannel, COUNT> channels;
};
```

每个通道包含：

| 字段 | 说明 |
| --- | --- |
| `name` | 通道名称，如“拉线 1” |
| `unit` | 工程单位，当前为 `mm` |
| `rawCounts` | SSI 原始计数值 |
| `displayValue` | 换算后的位移值 |
| `valid` | 数据是否有效 |

### 9.2 传感器寄存器映射

BRT38 SSI 拉线传感器通过 PLC 映射到 Modbus 保持寄存器。

| 通道 | 寄存器范围 | 默认名称 |
| --- | --- | --- |
| 1 | MW256 - MW257 | 拉线 1 |
| 2 | MW258 - MW259 | 拉线 2 |
| 3 | MW260 - MW261 | 拉线 3 |
| 4 | MW262 - MW263 | 拉线 4 |
| 5 | MW264 - MW265 | 拉线 5 |
| 6 | MW266 - MW267 | 拉线 6 |

每路传感器占 2 个 16 位寄存器，低字在前，高字在后，合成为 32 位有符号计数值。

### 9.3 位移换算

当前代码使用如下换算参数：

```cpp
kSpoolCircumference = 250.0f;
kSingleTurnRes = 4096.0f;
```

换算公式：

```text
displayValue(mm) = rawCounts * (spoolCircumference / singleTurnResolution)
```

即当前默认比例约为：

```text
250 / 4096 = 0.061035 mm/count
```

### 9.4 SensorPanel

`SensorPanel` 是右侧 6 路传感器数据显示面板。

显示内容：

- 通道名称
- 实时数值
- 单位

当 PLC 未连接或数据无效时，数值显示为 `---`。

## 10. 寄存器映射说明

寄存器地址统一放在 `communication/RegisterMap.h` 中。

主要区域：

| 区域 | 说明 |
| --- | --- |
| 轴控制区 | 每轴占用固定数量寄存器，用于控制字、目标速度、目标位置、状态字、实际速度、实际位置、转矩、故障码 |
| 传感器区 | BRT38 SSI 6 路传感器数据，起始地址为 MW256 |
| 系统状态区 | 系统状态、报警、急停等全局寄存器 |

本项目中本地扩展轴使用轴号 6 和 7，轮询时一次读取两个轴的数据块。

## 11. 线程与信号槽

### 11.1 PLC 线程

```text
MainWindow
└── QThread m_plcThread
    └── PlcWorker
        ├── ModbusTcpClient
        ├── QTimer m_pollTimer
        └── QTimer m_reconnTimer
```

PLC 线程负责所有 Modbus 读写操作。

### 11.2 平台线程

```text
MainWindow
└── QThread m_platformThread
    └── PlatformWorker
        ├── UdpPlatformClient
        └── QTimer m_watchdog
```

平台线程负责 UDP socket 绑定、命令发送和推送帧解析。

### 11.3 UI 更新方式

Worker 读取到数据后通过信号发送给 UI：

| 信号 | 接收者 | 说明 |
| --- | --- | --- |
| `motorDataUpdated` | `MotorPanel::onMotorDataUpdated` | 更新本地扩展轴状态 |
| `sensorDataUpdated` | `SensorPanel::onSensorDataUpdated` | 更新传感器显示 |
| `platformStateUpdated` | `PlatformPanel::onPlatformStateUpdated` | 更新平台状态 |
| `connectionStateChanged` | `MainWindow::onPlcConnectionChanged` | 更新 PLC 连接状态 |
| `connectionChanged` | `MainWindow::onUdpConnectionChanged` | 更新 UDP 在线状态 |
| `errorOccurred` | `MainWindow::onError` | 状态栏显示错误信息 |

## 12. 构建与运行

### 12.1 依赖环境

- Qt 5 或 Qt 6
- CMake 3.16 及以上
- C++17 编译器
- Windows 下可使用 MSVC 2022

### 12.2 CMake 构建

常规构建流程：

```bash

cmake --build build --target DeckLab
```

当前工程中已有 Qt Creator 生成的构建目录：

```text
build/Desktop_Qt_6_8_3_MSVC2022_64bit-Debug/
```

## 13. 运行流程

1. 启动程序。
2. 在顶部 PLC 区输入 PLC IP、端口和从站号。
3. 点击连接 PLC。
4. PLC 连接成功后，左侧电机面板和右侧传感器面板开始刷新。
5. 在顶部 Stewart UDP 区输入平台控制器 IP 和命令端口。
6. 点击连接平台。
7. 平台推送帧正常接收后，中间平台面板显示在线状态和实时位姿。
8. 根据需要执行上电、模式设置、目标位姿发送、停止等操作。

## 14. 安全注意事项

- 急停按钮会向所有轴写停止控制字，并写入系统急停命令寄存器。
- 速度和位置指令应在设备使能、无故障、现场安全确认后发送。
- 实际平台联调时，建议先使用低速、小位移目标进行测试。
- 修改寄存器地址前必须确认 PLC 端变量映射与 `RegisterMap.h` 保持一致。
- UDP 目标位姿范围应结合平台机械限位和厂家软件参数设置，不应只依赖 UI 输入范围。
